Base URL

Base URL 
https://api.chatflowai.com/v1

Rate Limits

API requests are limited to:

  • Starter Plan: 1,000 requests per hour
  • Professional Plan: 10,000 requests per hour
  • Enterprise Plan: 100,000 requests per hour

Authentication

The ChatFlow AI API uses API keys for authentication. Include your API key in the request headers.

Authentication Header 
Authorization: Bearer YOUR_API_KEY

Getting Your API Key

  1. Sign up for a ChatFlow AI account
  2. Go to your dashboard settings
  3. Navigate to the "API Keys" section
  4. Generate a new API key

Quick Start Guide

Get started with the ChatFlow AI API in just a few steps:

1. Create a Chatbot

JavaScript 
const response = await fetch('https://api.chatflowai.com/v1/chatbots', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'My First Chatbot',
    description: 'A helpful customer service bot',
    industry: 'E-commerce'
  })
});

const chatbot = await response.json();
console.log('Chatbot created:', chatbot.id);

2. Upload Training Data

JavaScript 
const formData = new FormData();
formData.append('file', trainingFile);
formData.append('chatbot_id', chatbot.id);
formData.append('content_type', 'faq');

const uploadResponse = await fetch('https://api.chatflowai.com/v1/upload-training-data', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  },
  body: formData
});

const uploadResult = await uploadResponse.json();
console.log('Upload status:', uploadResult.status);

3. Start Training

JavaScript 
const trainingResponse = await fetch('https://api.chatflowai.com/v1/training/start', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    chatbot_id: chatbot.id,
    job_type: 'initial_training'
  })
});

const trainingJob = await trainingResponse.json();
console.log('Training started:', trainingJob.job_id);

Data Ingestion APIStable

Upload and process training data for your chatbots.

POST /upload-training-data
Upload training data files (PDF, CSV, JSON, DOCX)

Parameters

Parameter Type Required Description
file File Yes Training data file
chatbot_id String Yes Target chatbot ID
content_type String Yes Type: faq, conversation, document, knowledge_base
title String No Optional title for the data
category String No Category for organization

Response

JSON Response 
{
  "status": "queued",
  "items_count": 25,
  "job_ids": ["uuid1", "uuid2"],
  "file_type": "text/csv"
}
GET /training-data/{chatbot_id}
Get training data status and metadata

Query Parameters

Parameter Type Default Description
page Integer 1 Page number for pagination
limit Integer 50 Items per page (max 100)

Training Pipeline APIStable

Manage chatbot training jobs and model versions.

POST /training/start
Start a new training job for a chatbot

Request Body

JSON Request 
{
  "chatbot_id": "uuid",
  "job_type": "initial_training",
  "config": {
    "batch_size": 16,
    "num_epochs": 3,
    "learning_rate": 2e-5,
    "validation_split": 0.2
  }
}

Response

JSON Response 
{
  "job_id": "training-job-uuid",
  "status": "queued",
  "estimated_duration": "15-30 minutes"
}

Vector Search APIStable

Perform semantic similarity search across your chatbot's knowledge base.

POST /search
Perform semantic similarity search

Request Body

JSON Request 
{
  "query": "How do I reset my password?",
  "chatbot_id": "uuid",
  "max_results": 5,
  "similarity_threshold": 0.8,
  "search_types": ["question", "answer", "knowledge"],
  "category_filter": "support"
}

Response

JSON Response 
{
  "results": [
    {
      "id": "result-uuid",
      "content": "To reset your password, click 'Forgot Password' on the login page...",
      "similarity_score": 0.95,
      "source_type": "faq",
      "metadata": {
        "title": "Password Reset Instructions",
        "category": "support",
        "tags": ["password", "login", "security"]
      }
    }
  ],
  "total": 1,
  "query": "How do I reset my password?",
  "threshold": 0.8
}
GET /search/hybrid/{chatbot_id}
Perform hybrid search across all content types

Query Parameters

Parameter Type Required Description
q String Yes Search query
user_id String No User ID for personalization

Best Practices

Data Quality

  • Ensure training data is clean and well-formatted
  • Use diverse examples for better generalization
  • Regular data quality assessments
  • Remove duplicate or irrelevant content

Performance Optimization

  • Use appropriate similarity thresholds (0.7-0.9)
  • Implement caching for frequent queries
  • Monitor response times and optimize accordingly
  • Use batch operations when possible

Security

  • Keep API keys secure and rotate regularly
  • Use HTTPS for all API requests
  • Implement proper error handling
  • Monitor API usage for anomalies

Troubleshooting

Common Issues

401 Unauthorized

Check that your API key is correct and included in the Authorization header.

Rate Limit Exceeded

Implement exponential backoff and respect rate limits. Consider upgrading your plan for higher limits.

Training Job Failed

Check the error message in the training job status. Common issues include insufficient training data or malformed input files.

Low Search Accuracy

Try lowering the similarity threshold, adding more diverse training data, or retraining your model.
